## Directory Purposes

**`.github/workflows/`:**
- Purpose: CI/CD automation
- Contains: GitHub Actions workflows
- Key files: `build-rag.yml` (triggers on content changes, runs tests, builds embeddings, uploads to R2)

**`content/`:**
- Purpose: Source documents for RAG knowledge base
- Contains: Plain-text files (.txt, .md) to be chunked and embedded
- Key files: `profile.txt` (3.1 KB), `resume.txt` (13.2 KB)
- Note: Changes here trigger the build-rag.yml workflow

**`journal/`:**
- Purpose: Generated blog post HTML files (one per post)
- Contains: Static HTML (5-15 KB each, pre-rendered from blog_posts.json)
- Generated by: `generate_blog.py`
- Not hand-edited; regenerate after updating blog_posts.json

**`rag/`:**
- Purpose: Compiled embeddings (gitignored, uploaded to Cloudflare R2)
- Contains: `documents.json` (chunk metadata), `vectors.json` (embeddings array)
- Generated by: `build_embeddings.py`
- Used by: Worker (fetches from R2 at runtime)
- Note: Not committed; R2 is source of truth for production

**`scripts/`:**
- Purpose: Deployment and utility helpers
- Key files:
  - `publish-worker.sh`: Wraps `wrangler deploy` with environment setup
  - `set-r2-secrets.sh`: Configures `CF_ACCOUNT_ID`, `CF_R2_ACCESS_KEY`, `CF_R2_SECRET_KEY` as GitHub secrets
  - `render-svg-to-png.mjs`: Converts SVG assets to PNG (manual social media images)

**`tests/`:**
- Purpose: Unit tests for Python and JavaScript
- Contains:
  - `test_build_embeddings.py`: Tests chunking, fake embeddings, determinism
  - (Worker tests are in `worker/rag-helpers.test.js`)

**`worker/`:**
- Purpose: Cloudflare Workers AI runtime code
- Key files:
  - `chat-worker.js`: HTTP handler, RAG orchestration
  - `rag-helpers.js`: Reusable vector/similarity functions
  - `rag-helpers.test.js`: Unit tests (run with `node --test`)
  - `wrangler.toml`: Configuration (AI binding, environment variables)

## Naming Conventions

**Files:**
- HTML pages: lowercase with hyphens (`index.html`, `resume.html`, `4everinbeta.html`)
- Python modules: lowercase with underscores (`build_embeddings.py`, `generate_blog.py`)
- JavaScript: lowercase with hyphens for bundles (`chat.js`, `chat-worker.js`)
- Test files: `test_*.py` (Python), `*.test.js` (JavaScript)
- Data files: lowercase with hyphens (`blog_posts.json`, `rag-helpers.js`)

**Directories:**
- Config: dot-prefix (`.github`, `.claude`, `.planning`)
- Feature areas: lowercase no-separator (`content`, `journal`, `worker`, `scripts`)
- Generated: explicit name (`rag/` for embeddings, `journal/` for posts)

**Variables & Functions:**
- Python: snake_case for functions and variables (`chunk_text`, `generate_embeddings`, `brandChatKnowledge`)
- JavaScript: camelCase for functions and variables (`cosineSimilarity`, `rankDocuments`, `handleChat`)
- Constants: UPPER_SNAKE_CASE in Python (`CHUNK_SIZE`, `FAKE_ENV_FLAG`)

## Where to Add New Code

**New Blog Post:**
1. Open `blog_posts.json`
2. Add entry to array with: `title`, `slug`, `pubDate` (RFC 2822), `description`, `content` (HTML), optional `heroIntro`
3. Run `python generate_blog.py`
4. Check generated `journal.html` and `journal/{slug}.html`
5. Commit blog_posts.json + generated journal files

**New Feature for Chat (Fallback Knowledge):**
1. Add keyword topic to `brandChatKnowledge` array in `chat.js` (line 10-41)
2. Format: `{ keywords: [...], response: "..." }`
3. Test locally by disabling `window.BRAND_CHAT_ENDPOINT` in index.html
4. For persistent knowledge, update `content/profile.txt` or `content/resume.txt` instead, regenerate embeddings, and rely on Worker

**New Feature for Worker (Remote Chat):**
1. Edit `worker/chat-worker.js`
2. Place request handling logic in `handleChat()` function (line 57+)
3. If new vector search logic needed, add to `worker/rag-helpers.js`
4. Add tests to `worker/rag-helpers.test.js`
5. Run `npm run test:worker` to verify
6. Deploy via `./scripts/publish-worker.sh`

**New Source Content for RAG:**
1. Add `.txt` or `.md` file to `content/` directory
2. Run `python build_embeddings.py` locally to test
3. Check `rag/documents.json` and `rag/vectors.json` are generated
4. Commit content file; GitHub Action will build and upload to R2

**New Utility Script:**
1. Place in `scripts/` directory
2. Make executable: `chmod +x script-name.sh` or `chmod +x script-name.mjs`
3. Add to README.md if user-facing
4. Call from manual process or CI/CD step as needed

**New Test:**
- Python unit tests: `tests/test_*.py`, use `unittest` framework
- JavaScript unit tests: `worker/*.test.js`, use Node.js native `test` module
- Run: `npm test` (both suites) or individually with `npm run test:python` / `npm run test:worker`

## Special Directories

**`rag/`:**
- Purpose: Compiled embeddings
- Generated: Yes (by build_embeddings.py)
- Committed: No (in .gitignore)
- Rebuilt: On every push to content/, build_embeddings.py, or tests/
- Deployed: To R2 via GitHub Action AWS CLI sync

**`journal/`:**
- Purpose: Generated blog post HTML
- Generated: Yes (by generate_blog.py)
- Committed: Yes (checked into git)
- Rebuilt: Manually after editing blog_posts.json
- Deployed: As part of GitHub Pages (static files)

**`.github/workflows/`:**
- Purpose: Automation definitions
- Generated: No (hand-edited)
- Committed: Yes
- Note: Main trigger is `build-rag.yml` on content changes

**`.planning/codebase/`:**
- Purpose: Architecture documentation (this output)
- Generated: Yes (by gsd-map-codebase agent)
- Committed: Yes
- Note: Reference for future phase planning and execution
